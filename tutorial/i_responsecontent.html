El contenido de la respuesta son los datos que un servidor web devuelve a la solicitud de un cliente. Ya ha sido tocado por la función _content utilizada en todos los ejemplos anteriores. Esta función es simple, solo genera el texto (o texto formateado) en el flujo de respuesta CWPX.

De forma predeterminada, la respuesta MIME de un script CWPX es "text/html". Esto se puede cambiar usando la función API iCWPX "_header("<headername>", "<mimetype>");". De hecho, la función "_header" funciona para obtener encabezados de solicitud y para establecer encabezados de respuesta. 

_header("<request headername>");
_header("<response headername>", "<header value>");

Si, por ejemplo, el tipo de respuesta debe cambiarse a "texto/plain", se haría de esta manera: 

_header("Content-Type", "text/plain");

Por defecto, el tipo de status code y descripción de status de un script CWPX es "200 OK". Esto se puede cambiar usando la función API de iCWPX "_status(<código de estado>, "<nombre de código de estado>");", por ejemplo: 

_status(404, "Not Found");

Cree una carpeta en el directorio web del servidor (www) llamada responsecontent y vaya a esa carpeta. Abra un editor de texto y cree un archivo llamado responsecontent.c, este será el controlador principal de la aplicación. El archivo actual responsecontent.c debe tener el siguiente contenido: 

#include <cwpx.h>
#include <icwpx.h>

void showPlainText(){
	_header("Content-Type", "text/plain");
	_content("<h3>This is a plain text response!</h3>");
}

void _http(){

	_maproute("/showmessage", showPlainText);
	
	_status(404, "Not Found");
	_content("<h3>The resource was not found in this app</h3>");
	
	return;
}

Compile la fuente: 

$ gcc responsecontent.c -I"directorio de cwpx.h" -o responsecontent.cwpx -L"directorio de cwpx.dll(.so) y icwpx.dll(.so)" -lcwpx -licwpx

Edite el archivo .htaccess para redirigir todas las solicitudes a "/responsecontent" hacia el archivo responsecontent.cwpx: 

<IfModule mod_rewrite.c>
 
	# Disable directory listings (mod_autoindex)
	Options -Indexes

	# Prevent mod_dir appending trailing slash to directories
	DirectorySlash Off
 
	Options +FollowSymLinks
	
	RewriteEngine On
	RewriteRule ^(.*)$ responsecontent.cwpx [QSA]

</IfModule>

Inicie un navegador web y vaya a "localhost/responsecontent/showmessage". Si todo está bien, debería ver un mensaje de texto plano o una respuesta de 'no encontrada' para cualquier otra URL.
