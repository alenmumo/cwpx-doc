typedef struct Config Config

Config object for getting and setting context data.

The Config object allows checking some configuration settings that are applied and implemented by the CWPX scripts. They're meant to be 'read-only', although some of them can also be modified at runtime. Functions apply these params:

Parameters
dirA path for the new directory.
idThe name for that identifies sessions' cookie.
sessdurDuration of sessions.
logfileNew path for logfile.
buf_sizeNew length for buffer size.
maxqueueelemsNew length for max. queue elements.
allowpostWhether or not allowing 'POST' requests.
maxpostNew length for max. 'POST' body.
Returns
A valid 'char *' containinig the value, NULL or 0|1 in case of returning 'int'.

◆ Cwpx_Args

typedef struct Cwpx_Args Cwpx_Args

Cwpx_Args object for getting and setting context data.

The Cwpx_Args object allows setting context variables that are shared across CWPX scripts in one request, by means of 'forwarding' or 'including'. The lifespan of the variables is just the lifespan of the current request. The 'set' and 'get' functions are not called directly, a macro replaces the call by 'set_context' and 'get_context'. These functions must be used carefully. The following piece of code is an example of usage:

int a = 99999;
set_context("a", &a, sizeof(int));
...
int *aPtr = get_context("a", sizeof(int));
int b = aPtr != NULL ? *aPtr : 0;

Functions apply these params:

Parameters
keyA name that identifies the requested data.
valueA value to be assigned.
size_ofSize of the value to be assigned.
vartypeDetermined vartype of the value.
Returns
A valid 'void *' containinig the value, NULL or 0|1 in case of returning 'int'.
Note
'vartype' is determined from 'size_of', they are tightly coupled.
Warning
Always try to use 'sizeof(sometypehere)' instead of providing the size manually, except for 'char *' values.

◆ Cwpx_Context

typedef struct Cwpx_Context Cwpx_Context

Cwpx_Context object for handling a HTTP request.

The Cwpx_Context holds all of the objects involved in handling a HTTP request, such as a request parser, a response setter, context arguments, configuration and sessions, as well as the entry points that the CWPX scripts use to interact with the framework.

◆ Response

typedef struct Response Response

Response object for writing response data.

The Response object provides functions for setting data to the HTTP response. The data can be sent in several ways, such as setting 'key-value' pairs for response headers or just printing some information in the response body.

◆ Session

typedef struct Session Session

Session object for getting and setting session data.

The Session object allows setting session variables that are shared across CWPX scripts in different requests. The lifespan of the variables is just the lifespan of the cookie that handles the session. The 'set' and 'get' functions are not called directly, a macro replaces the call by 'set_session' and 'get_session'. These functions must be used carefully. The following piece of code is an example of usage:

session_start();
int a = 678;
set_session("a", &a, sizeof(int));
...
int *aPtr = get_session("a", sizeof(int));
int b = aPtr != NULL ? *aPtr : 0;
