Detailed Description

File containing example of doxygen usage for quick reference.

Author
TeamCWPX
Date
17 Aug 2022 This header defines the iCWPX web framework interface for running CGI scripts in C language on top of the classical CWPX interface, for using mainstream-like concepts such as friendly URLs, MVC and REST patterns.

Macro for emulating 'annotations' to be used as special instructions.

The '_' macro is used as a shortcut of the 'icwpx_addat' function, which receives the name of an annotation, a string representing the type of variable, a string for the variable name and the value of the variable. Those elements are pushed into a parameters queue that will be consumed by the '_p' function. The macro shortens that for the user just sends the annotation name, the type (no string), and the variable name. The variable's value is filled later.

_("@path", int, x);
Parameters
atName of the origin where a parameter should be extrated from.
vartypeType of the variable where the parameter should be put after extrated.
varnameName of the variable where the parameter should be put after extrated.

Map a URL pattern to be served by a function.

The '_maproute' macro is used as a shortcut of the 'icwpx_maproute' function, which takes a URL pattern and a function name. When 'icwpx_maproute' is called, it evaluates that the pattern and the request URI match and then call the specified function.

void myFunction();
...
_maproute("/url/pattern", myFunction);
Parameters
XA URL pattern to be mapped.
YA function name for serving the request.
Returns
If found, the script returns right after the mapped function ends its execution, if not, the function returns -1 and the script will continue evaluating other functions.
